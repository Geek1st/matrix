## 🔴 高优先级缺失项（必须补齐）

### 功能完整性

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **特殊宝石** | 仅 6 种基础宝石，无 4/5 匹合特殊效果 | 添加条纹宝石（4 匹合）、彩虹炸弹（5 匹合），提升策略深度 |
| **关卡目标** | 仅“存活 30 步”模式，无明确目标 | 添加分数目标、收集宝石、清除障碍等目标类型 |
| **提示系统** | `DeadlockChecker.findValidMoves()` 已存在但未暴露 | 添加“提示”按钮，高亮一个合法走法 |
| **分数弹出** | 仅 HUD 更新，无即时反馈 | 在匹配位置显示浮动“+100”文本，向上淡出 |
| **连击动画** | 仅文字“X COMBO!”，无视觉冲击 | 添加粒子爆发、屏幕震动或扩散光环效果 |
| **音效** | 完全静默 | 添加 Web Audio API 音效：交换、匹配、连击、游戏结束 |

### 视觉与 UX

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **交换动画** | 瞬间交换，仅 150ms 延迟 | 实现平滑位置插值（200-300ms ease-out） |
| **下落动画** | 宝石瞬间重定位 | 添加重力动画 + 弹跳缓动 |
| **匹配爆炸** | 仅淡出 40% 透明度 | 添加缩放 + 淡出 + 粒子爆发效果 |
| **合法走法高亮** | 玩家需猜测哪些走法有效 | 高亮可形成匹配的宝石（脉冲或发光） |
| **无效交换反馈** | 无效交换静默回退 | 添加震动动画、红色闪烁或“X”图标 |
| **交互式教程** | 仅状态栏文字提示 | 添加 2-3 步引导：高亮第一个宝石、显示滑动方向、引导完成首次匹配 |
| **响应式布局** | 固定 6x6 80px，无缩放 | 根据视口缩放，保持比例，移动端增大触摸目标 |
| **移动端触摸优化** | 触摸目标可能过小 | 增大单元格尺寸，添加触摸反馈涟漪，确保 44px 最小触摸目标 |
| **无障碍支持** | 无 ARIA 标签、无语义 HTML | 添加 `role="application"`、宝石 `aria-label`、分数/步数变化播报 |
| **色盲模式** | 宝石仅靠颜色区分 | 为每种宝石添加图案（条纹、点、十字等） |

### 稳定性

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **级联无限循环风险** | `processCascade()` 无安全计数器 | 添加 `cascadeIterations < 100` 上限 |
| **异常状态锁定** | `attemptSwap()` 若抛异常，`isProcessing` 永为 `true` | 添加 `try { ... } finally { isProcessing = false; }` |
| **输入无反馈** | `isProcessing` 期间忽略输入，无视觉提示 | 显示“处理中…”或禁用光标 |
| **会话持久化** | 页面刷新后进度丢失 | `localStorage` 保存分数、步数、棋盘序列化 |

### 发布就绪性

| 缺失项 | 说明 | 建议 |
|--------|------|------|
| **用户指南** | 仅一行状态栏提示 | 创建 `demo-README.md`：控制说明、规则、计分、连击机制、故障排除 |
| **独立打包脚本** | 无分发单文件脚本 | 创建 `scripts/package-demo.ps1`：内联 CSS/JS、优化、输出 `dist/demo-v1.0.0.html` |
| **浏览器兼容性清单** | 无测试清单 | 创建 `BROWSER_TEST.md`：Chrome/Edge/Firefox/Safari、移动端已知问题 |
| **移动端测试清单** | 无移动设备测试清单 | 创建 `MOBILE_TEST.md`：触摸响应、视口缩放、性能 |
| **截图** | 无视觉文档 | 拍摄 3-5 张：初始棋盘、连击中局、游戏结束、移动端视图 |
| **演示视频** | 无动画演示 | 录制 30-60 秒：基础匹配、连击级联、死锁洗牌、游戏结束 |

| **宝石发光/光环** | 平面颜色，添加脉冲发光效果 |

| **生成动画** | 新宝石瞬间出现，添加淡入 + 缩放动画 |
| **进度指示器** | 仅步数计数，添加进度条或星级预览 |
| **暂停菜单** | 无暂停功能，添加暂停按钮：继续/重启/设置/退出 |

| **高对比度模式** | 低对比度背景，添加高对比度颜色切换 |

### 稳定性

| 缺失项 | 说明 |
|--------|------|

| **输入队列** | 缓存最后一次有效输入，`isProcessing` 结束后执行 |

## 🎯 推荐实施顺序

### Phase 1: 核心反馈与稳定性（1-2 周）

1. **级联安全计数器** — 防止死循环
2. **异常保护** — `try/finally` 包裹关键操作
3. **交换动画** — 平滑位置插值
4. **下落动画** — 重力 + 弹跳
5. **匹配爆炸效果** — 粒子 + 缩放淡出
6. **音效** — Web Audio API 基础音效
7. **分数弹出** — 浮动文本反馈
8. **连击动画** — 粒子爆发或屏幕震动
9. **合法走法高亮** — 提示可匹配宝石
10. **提示按钮** — 暴露 `findValidMoves()`

### Phase 2: UX 基础（1 周）

11. **交互式教程** — 2-3 步引导
12. **响应式布局** — 视口缩放
13. **移动端触摸优化** — 增大目标、触摸反馈
14. **无障碍支持** — ARIA 标签、屏幕阅读器
15. **色盲模式** — 宝石图案区分
16. **会话持久化** — localStorage 保存进度

### Phase 3: 功能深度（1-2 周）

17. **特殊宝石** — 4 匹合条纹、5 匹合彩虹
18. **关卡目标** — 分数目标、收集宝石
19. **道具系统** — 锤子、洗牌、炸弹
20. **关卡进度** — 5-10 个演示关卡

### Phase 4: 发布准备（1 周）

21. **用户指南** — `demo-README.md`
22. **独立打包脚本** — `scripts/package-demo.ps1`
23. **浏览器兼容性清单** — `BROWSER_TEST.md`
24. **移动端测试清单** — `MOBILE_TEST.md`
25. **截图** — 3-5 张游戏画面
26. **演示视频** — 30-60 秒录制

## 📝 总结

当前 `demo.html` 已验证核心三消算法，但缺少“游戏感”（视觉/音频反馈）和“产品感”（引导、目标、进度）。要达到真正演示级别，需优先补齐 Phase 1 的 10 项核心反馈与稳定性改进，再逐步推进 UX、功能深度和发布准备。

**最小可行演示升级**：完成 Phase 1 + Phase 2 前 5 项，即可从“算法原型”转变为“可玩演示”，适合内部评审或用户测试。
