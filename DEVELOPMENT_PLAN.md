## 📋 任务分解与依赖关系

### Phase 1: 核心反馈与稳定性（关键路径）

#### Task Group A: 稳定性基础（必须最先完成）
- **A1**: 级联安全计数器 — `processCascade()` 添加 `cascadeIterations < 100`
- **A2**: 异常保护 — `attemptSwap()` 添加 `try/finally`
- **A3**: 输入反馈 — `isProcessing` 时显示处理中状态

**依赖**: 无前置依赖，可立即执行
**预估**: 0.5 天

#### Task Group B: 动画系统（依赖 A 完成）
- **B1**: 交换动画 — 平滑位置插值（200-300ms ease-out）
- **B2**: 下落动画 — 重力 + 弹跳缓动
- **B3**: 匹配爆炸效果 — 粒子 + 缩放淡出

**依赖**: A1, A2 完成（避免动画与异常处理冲突）
**预估**: 1.5 天

#### Task Group C: 音效系统（独立模块）
- **C1**: Web Audio API 基础架构 — 音效管理器
- **C2**: 交换音效 — whoosh/click
- **C3**: 匹配音效 — pop/chime（连击升调）
- **C4**: 连击音效 — 递进式音效
- **C5**: 游戏结束音效 — game over jingle

**依赖**: 无前置依赖，可与 A 并行
**预估**: 1 天

#### Task Group D: 反馈系统（依赖 B 完成）
- **D1**: 分数弹出 — 浮动文本 + 向上淡出
- **D2**: 连击动画增强 — 粒子爆发或屏幕震动
- **D3**: 合法走法高亮 — 脉冲/发光提示
- **D4**: 提示按钮 — 暴露 `findValidMoves()`

**依赖**: B1, B2 完成（动画系统就绪后再添加反馈）
**预估**: 1 天

---

### Phase 2: UX 基础（可部分并行）

#### Task Group E: 教程与引导（依赖 D 完成）
- **E1**: 交互式教程 — 2-3 步引导流程
- **E2**: 无效交换反馈 — 震动 + 红色闪烁

**依赖**: D3, D4 完成（提示系统就绪后再做教程）
**预估**: 1 天

#### Task Group F: 响应式与无障碍（独立模块）
- **F1**: 响应式布局 — 视口缩放 + 保持比例
- **F2**: 移动端触摸优化 — 增大目标 + 触摸反馈
- **F3**: 无障碍支持 — ARIA 标签 + 屏幕阅读器
- **F4**: 色盲模式 — 宝石图案区分

**依赖**: 无前置依赖，可与 Phase 1 并行
**预估**: 1.5 天

#### Task Group G: 会话持久化（独立模块）
- **G1**: localStorage 保存 — 分数、步数、棋盘序列化
- **G2**: 进度恢复 — 页面加载时恢复状态

**依赖**: 无前置依赖，可并行
**预估**: 0.5 天

## 🔧 技术债务与风险

### 技术债务
- 当前使用 `setTimeout` 做动画，应迁移到 `requestAnimationFrame`
- 硬编码游戏参数，应提取到配置文件
- 无单元测试，后续需补充

### 风险
- 动画系统可能与现有逻辑冲突 → 先在分支开发，充分测试后合并
- 音效系统需要音频资源 → 使用免费音效或占位音效
- 特殊宝石逻辑复杂 → 先实现最简版本，迭代优化
